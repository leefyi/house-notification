# ADR-0001: 技术选型

## 状态

已接受

## 背景

需要构建一个定时监控深圳市住建局网站保租房/公租房公告的程序，并通过手机推送通知用户。

## 决策

### 1. 运行环境：GitHub Actions（非本地 Mac）

**选择：** GitHub Actions 定时任务

**备选：**
- Mac 本地 launchd — 依赖本机开机，合盖即停
- 云服务器 — 需付费 30-50 元/月

**理由：** 免费、不依赖本机状态、稳定运行。private repo 可保护代码和配置。

### 2. 通知方式：Bark（非短信）

**选择：** Bark iOS App 推送

**备选：**
- 短信 API — 需企业资质，个人难申请，0.04-0.05 元/条
- 邮件 — 延迟高，不够即时
- 企业微信 webhook — 需企业微信

**理由：** 免费、即时推送、个人可用、5 分钟接好。用户使用 iPhone。

### 3. 状态存储：GitHub Actions cache（非 commit）

**选择：** GitHub Actions cache 存储已见 URL 集合

**备选：**
- Commit JSON 到 repo — 用户不想处理 git 冲突
- GitHub Gist — 多一个服务
- 外部数据库 — 过度设计

**理由：** 零 commit、零冲突。7 天过期机制在每日运行下不会触发。唯一风险是停用超 7 天会丢失状态（无害，最多重新通知一遍旧公告）。

### 4. 新公告检测：时间过滤 + URL 去重

**选择：** 双重机制 — 记录上次抓取时间过滤范围 + 记录所有已见 URL 去重

**理由：** 时间过滤缩小扫描范围，URL 去重兜底防止重复通知。

### 5. 语言：Python

**选择：** Python

**备选：** Node.js/TypeScript

**理由：** 爬虫生态最成熟（requests/BeautifulSoup），脚本简洁，GitHub Actions 原生支持。
